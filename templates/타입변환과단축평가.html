9장 타입 변환과 단축 평가

9.1 타입 변환이란?

개발자가 의도적으로 타입을 변환 => 명시적 타입 변환
자바스크립트 엔진에 의해 자동 변환 => 암묵적 타입 변환
암묵적 타입 변환이 어떠한 경우 발생하는지 알고 있어야 오류를 줄이거나, 코드를 편하게 짤 수 있다.
가독성은 안 좋아질 가능성이 높다.

9.2 암묵적 타입 변환

9.2.1 문자열 타입으로 변환

문자 + 다른타입은 거의 문자타입으로 변환된다.
예외 -> symbol타입

9.2.2 숫자 타입으로 변환

산술연산자와 비교 연산자는 숫자값을 만들거나 비교하기 위해 만들어진것이므로 숫자가 아닌 타입은 숫자로 형 변환 된다.

9.2.3 불리언 타입으로 변환

false로 변환되는 경우만 알고 있으면 됨.

9.3 명시적 타입 변환

암묵적 타입 변환을 시키는것 보단 명시적으로 타입 변환을 시키는게 팀작업과 유지보수에 좋음.

9.3.1 문자열 타입으로 변환

String(값), 값.toString(), 값 + ''

9.3.2 숫자 타입으로 변환

Number(값), parseInt(값)
Number() 와 parseInt의 차이

9.3.3 불리언 타입으로 변환

Boolean(값), !!값

9.4 단축평가

9.4.1 논리 연산자를 사용한 단축 평가

단축 평가 표현식 평가 결과
true || anything true
false || anything anything
true && anything anything
false && anything false

9.4.2 옵셔널 체이닝 연산자

좌항이 null 이거나 undefined면 좌항을 아니면 우항의 프로퍼티 참조를 이어간다.

9.4.3 null 병합 연산자

값1 ?? 값2 에서 값1이 null이거나 undefined면 값2를 반환, 그 외 값1을 반환.
논리 연산자의 단축평가보다는 옵셔널 과 null 병합을 쓰는게 좋다.
